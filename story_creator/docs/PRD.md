# 故事创作引擎 (Story Creator) - 产品PRD

## 1. 产品概述

### 产品定位
**对话驱动的互动故事创作引擎** —— 通过AI引导，帮助有想法但文字能力弱的用户，将想象转化为完整小说、游戏策划案、剧本或自传。

### 核心价值
- 降低创作门槛：让任何人都能完成自己的故事
- 沉浸式体验：像玩游戏一样创作故事
- 多形态输出：小说、游戏策划案、剧本、自传

### 目标用户
- 想写自传的成功人士（文笔不够好）
- 想体验游戏剧情的玩家（输出成品可作为游戏设计参考）
- 有创作想法但写作能力欠缺的人

## 2. 功能架构

### 2.1 核心流程（5阶段）

```
┌─────────────────────────────────────────────────────────────────┐
│  阶段1        阶段2        阶段3        阶段4        阶段5      │
│  构思        框架          章节规划    章节创作    最终成品    │
│  引导        生成          详细规划    迭代完善    导出        │
└─────────────────────────────────────────────────────────────────┘
        ↑              ↑            ↑           ↑
      进度: 10%      30%          60%         90%
```

### 阶段1：构思引导
- **输入**：用户仅提供一个模糊想法（一句话描述）
- **AI行为**：通过多轮提问，明确用户意图
  - 题材类型（都市、玄幻、悬疑、科幻、自传等）
  - 核心冲突/主题
  - 主角特征
  - 期望结局
- **输出**：《创作意图确认书》

### 阶段2：框架生成
- **输入**：创作意图
- **AI行为**：生成完整世界观、角色设定、剧情大纲
  - 世界观/时代背景
  - 角色卡片（至少3个核心角色）
  - 角色关系图谱
  - 故事主线
  - 关键事件列表（5-8个里程碑）
  - 章节总数规划
- **输出**：《故事框架文档》，用户可修改确认

### 阶段3：章节规划
- **输入**：故事框架
- **AI行为**：为每个章节生成
  - 章节标题
  - 章节摘要（100-200字）
  - 预计字数
  - 本章核心事件
  - 涉及角色/场景
- **输出**：《章节规划表》，用户可调整顺序/增删

### 阶段4：章节创作（增强版）
- **输入**：选定的章节规划
- **AI行为**：
  1. **方向建议**：每次生成前，提供 3 个不同的故事发展方向
  2. **内容生成**：用户选择方向后，生成详细、丰富的章节内容
  3. **多轮互动**：每章节至少 3 轮互动（方向选择 → 生成 → 再次选择 → 继续生成）
- **输出**：《完整章节》，状态标记为"已完成"

### 2.2 输出形态
支持多种导出格式：
- 📖 **小说**：Markdown/PDF/TXT
- 🎮 **游戏策划案**：JSON结构化数据（含剧情树、角色数值、关卡设计）
- 🎬 **剧本**：标准剧本格式
- 📝 **自传**：第一人称叙事

## 4. 技术架构

### 4.1 技术栈

```
前端：Vue3 + Pinia + Vue Router (单HTML文件)
后端：FastAPI + 内存存储
AI：大模型对话接口（千帆/其他）
```

### 4.2 数据模型

```python
# 项目 Project
{
    "id": "uuid",
    "title": "故事标题",
    "idea": "用户原始想法",
    "type": "小说/游戏策划/剧本/自传",
    "status": "构思中/框架生成中/框架已生成/创作中/已完成",
    "progress": 0-100,
    "framework": {},
    "created_at": "timestamp",
    "updated_at": "timestamp"
}

# 角色 Character
{
    "id": "uuid",
    "project_id": "uuid",
    "name": "角色名",
    "role": "主角/配角/反派",
    "description": "角色描述",
    "appearance": "外貌",
    "personality": "性格",
    "background": "背景故事",
    "relationships": [{"target_id": "uuid", "type": "朋友/敌人/恋人"}]
}

# 章节 Chapter
{
    "id": "uuid",
    "project_id": "uuid",
    "order": 1,
    "title": "章节标题",
    "summary": "章节摘要",
    "content": "章节正文",
    "status": "待创作/进行中/已完成",
    "word_count": 0,
    "interaction_count": 0,  # 互动次数
    "direction_history": []  # 方向选择历史
}

# 章节创作方向 (Chapter Direction)
{
    "id": "uuid",
    "chapter_id": "uuid",
    "direction": "方向描述",
    "content": "生成的内容",
    "order": 1  # 第几次互动
}
```

## 6. 章节创作增强 - 详细设计

### 6.1 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│  用户点击章节 → 显示章节详情                                     │
│       ↓                                                        │
│  用户点击「AI生成内容」                                         │
│       ↓                                                        │
│  后端生成 3 个方向建议                                           │
│       ↓                                                        │
│  前端显示方向选择弹窗                                           │
│       ↓                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 选项1: 情感向     选项2: 悬疑向     选项3: 冒险向        │   │
│  └─────────────────────────────────────────────────────────┘   │
│       ↓                                                        │
│  用户选择方向                                                   │
│       ↓                                                        │
│  后端根据选择生成详细内容                                       │
│       ↓                                                        │
│  显示生成的内容 + 询问是否继续                                  │
│       ↓                                                        │
│  如果 interaction_count < 3，返回方向选择                       │
│  否则标记为「已完成」                                          │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 API 设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/chapters/{id}/directions` | POST | 获取方向建议（返回3个方向） |
| `/chapters/{id}/generate` | POST | 根据选择的方向生成内容 |
| `/chapters/{id}` | GET | 获取章节详情（包括已生成的内容） |

## 7. 验收标准

### MVP验收

- [ ] 用户可以输入一句话想法创建新项目
- [ ] AI通过对话引导明确创作意图
- [ ] 自动生成世界观、角色、剧情大纲
- [ ] 角色卡片正确展示信息
- [ ] 关系图谱展示角色关系
- [ ] 章节规划显示目录+摘要
- [ ] 可以选择章节进行创作
- [ ] AI生成章节正文
- [ ] 进度条实时更新
- [ ] 可以导出为Markdown

### 章节创作增强验收

- [ ] 点击「AI生成内容」显示3个方向选项
- [ ] 每个方向有明确的标题和描述
- [ ] 选择方向后生成详细的章节内容
- [ ] 每章节至少进行3轮方向选择互动
- [ ] 可以查看方向选择历史
- [ ] 章节内容在每次互动后累积丰富
